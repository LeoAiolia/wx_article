# LongCat：美团开源的大模型，到底什么来头？


---

## 一、LongCat 是什么

LongCat 是**美团**自研的大模型系列，采用 MoE（混合专家）架构，总参数规模 1.6 万亿，每 token 激活约 480 亿参数。

这个名字听起来有点互联网——美团的命名风格一贯如此——但它的技术规格一点也不含糊。接下来我们从几个关键维度来认识它。

---

## 二、核心能力

### 1. 百万级上下文

LongCat 原生支持 **100 万 token 超长上下文**。它采用了 LSA（Latent Sparse Attention）稀疏注意力机制，能够智能筛选关键信息，将超长文本的计算量降至线性级。

这意味着你可以把一整个项目的代码库、一本几百页的技术文档、或者几十轮的会议纪要丢给它，它仍然能保持对话连贯性。

对开发者来说，这个能力很实际：不用再把大文件拆成一块一块喂给模型，也不用担心说到后面忘了前面。

### 2. 编程与 Agent 能力

LongCat 在编程任务和智能体（Agent）场景中表现突出。它采用了「零计算专家机制」和 ScMoE 架构，推理速度明显更快，尤其适合耗时较长的复杂 Agent 应用。

简单说：它不只是回答问题，还能像 Agent 一样自主规划、调用工具、多步执行。

### 3. 是否支持生成图片

**不支持。** LongCat 是纯语言模型（Text-only），专注于文本推理和生成，不具备图片生成或多模态理解能力。它的技术路线围绕稀疏注意力、MoE 架构和国产算力优化展开，定位非常明确——做好文本和代码场景。

如果你需要图片生成，得看其他模型（如美团内部的其他产品线，或市面上的 FLUX、DALL·E 等）。

### 4. 国产算力驯服

这是 LongCat 最被技术圈讨论的一点：它是**业界首个在五万卡国产算力集群上完成全流程训练与推理的万亿参数模型**。

这意味着从训练到推理，整条链路不依赖英伟达 GPU。在当下的产业环境里，这件事的战略意义可能比模型本身的跑分更重要。

---

## 三、开源情况

LongCat 已经在 **GitHub** 和 **Hugging Face** 平台同步开源。开发者可以自行下载模型权重，也可以基于它做微调和二次开发。

对于不想自己部署的开发者，也可以通过官网 API 直接调用。

---

## 四、网上怎么评价

### 技术社区的声音

知乎等社区对 LongCat-2.0 的关注点主要集中在：

> 「LongCat-2.0 的意义可能不在于它是不是当下最强的模型，而在于它完整跑通了国产算力大规模训练的链路。」

这被认为是国产大模型基础设施建设的一个标志性事件。

### 排名参考

LongCat-2.0 的测试版本在 OpenRouter 平台上排名靠前，说明它在实际使用场景中具备一定竞争力。

### 客观来看

作为一个相对较新的模型系列，LongCat 还在快速迭代中。它的长上下文和 Agent 能力是差异化的卖点，而国产算力全链路打通这件事，让它在大模型国产化的讨论中占据了一个特殊位置。

---

## 五、怎么体验

如果你想亲自试试 LongCat，有几个渠道：

| 渠道 | 地址 | 说明 |
|---|---|---|
| 官网对话 | [longcat.chat](https://longcat.chat) | 直接网页对话体验 |
| API 开放平台 | 官网内申请 | 注册获取 API Key，集成到自己的应用 |
| GitHub | GitHub 搜索 LongCat | 开源模型权重和代码 |
| Hugging Face | Hugging Face 搜索 LongCat | 模型下载和体验 |

建议从 **longcat.chat** 开始，打开就能用，无需注册或配置。

## 结语

LongCat 的出现，让国产大模型阵营又多了一个有分量的选手。

它的长上下文能力适合真实项目场景，Agent 能力让它能处理更复杂的任务，而国产算力全链路的打通则超出了单纯的技术层面。

如果你对大模型在开发场景的应用感兴趣，LongCat 值得花点时间了解。
